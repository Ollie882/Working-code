# MediHealth Appointments

A simple, accessible **web app** for recording and viewing patient appointments for **MediHealth**, built as part of the **T Level Digital Production, Design & Development (Occupational Specialism)**.

This project uses **Supabase** for the database, a **W3C-compliant HTML form**, **CSS** for an app-like UI, and **Vanilla JavaScript** to connect the form to Supabase via PostgREST.

---

## ✨ Features

- Create appointments via a clean, accessible HTML form.
- Store data in **Supabase Postgres** (via PostgREST).
- List all appointments at the bottom of the page.
- App-style UI (mobile-friendly, high-contrast, visible focus).
- No frameworks required.

---

## 🧱 Tech Stack

- **Frontend:** HTML5, CSS3, Vanilla JS (ES6+)
- **Database/Backend:** Supabase (Postgres), PostgREST
- **Dev/Hosting:** Local (file or VS Code Live Server), GitHub Pages compatible for static hosting

---

## 🏗️ Architecture

```
index.html
```

## 🗄️ Database Schema (Supabase)

Run this SQL in the **Supabase SQL editor** to create the table:

```sql
CREATE TABLE IF NOT EXISTS tbl_customer_appointments (
  appointment_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  patient_name TEXT NOT NULL,
  patient_dob DATE NOT NULL,
  patient_telephone TEXT NOT NULL,
  patient_address TEXT NOT NULL,
  patient_medical_history TEXT NOT NULL,
  patient_next_of_kin TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);
```

> **Note:** Postgres stores **unquoted identifiers** in lowercase. Use `tbl_customer_appointments` in REST paths.

## 🔐 Row Level Security (RLS) & Policies

Enable **RLS** and add development policies to allow anonymous insert/select via the **publishable (anon) key**. For production, tighten these policies.

```sql
ALTER TABLE tbl_customer_appointments ENABLE ROW LEVEL SECURITY;

-- Dev/demo: allow select to anon
CREATE POLICY "Allow select to anon"
ON tbl_customer_appointments
FOR SELECT
TO anon
USING (true);

-- Dev/demo: allow insert to anon
CREATE POLICY "Allow insert to anon"
ON tbl_customer_appointments
FOR INSERT
TO anon
WITH CHECK (true);
```

### JavaScript (constants + submit handler + list rendering)

> Update `API_URL` and `API_KEY` with your Supabase project details.

```javascript
// Supabase project settings
const API_URL = "https://YOUR-PROJECT.supabase.co";
const API_KEY = "sb_publishable_XXXXXXXX";

// Table name
const APPOINTMENTS_TABLE = "tbl_customer_appointments";

document.addEventListener("DOMContentLoaded", () => {
  const form = document.getElementById("appointment-form");
  if (!form) return;

  form.addEventListener("submit", async (event) => {
    event.preventDefault();
    await addAppointment();
    await loadAppointments(); // refresh list after insert
  });

  loadAppointments(); // initial render
});

async function addAppointment() {
  const body = {
    patient_name: document.getElementById("patient_name").value.trim(),
    patient_dob: document.getElementById("patient_dob").value.trim(),
    patient_telephone: document.getElementById("patient_telephone").value.trim(),
    patient_address: document.getElementById("patient_address").value.trim(),
    patient_medical_history: document.getElementById("patient_medical_history").value.trim(),
    patient_next_of_kin: document.getElementById("patient_next_of_kin").value.trim()
  };

  // Basic validation
  if (!body.patient_name || !body.patient_dob || !body.patient_telephone || !body.patient_address) {
    alert("Please fill in patient name, date of birth, telephone, and address.");
    return;
  }

  // POST to Supabase
  const res = await fetch(`${API_URL}/rest/v1/${APPOINTMENTS_TABLE}`, {
    method: "POST",
    headers: {
      apikey: API_KEY,
      Authorization: `Bearer ${API_KEY}`,
      "Content-Type": "application/json",
      Prefer: "return=minimal"
    },
    body: JSON.stringify(body)
  });

  if (!res.ok) {
    console.error(await res.text());
    alert("Could not add appointment.");
    return;
  }

  alert("Appointment booked successfully!");

  // Clear form
  ["patient_name","patient_dob","patient_telephone","patient_address","patient_medical_history","patient_next_of_kin"]
    .forEach(id => document.getElementById(id).value = "");
}

async function loadAppointments() {
  const list = document.getElementById("appointments-list");
  if (!list) return;

  const res = await fetch(
    `${API_URL}/rest/v1/${APPOINTMENTS_TABLE}?select=*&order=created_at.desc`,
    { headers: { apikey: API_KEY, Authorization: `Bearer ${API_KEY}` } }
  );

  if (!res.ok) {
    list.innerHTML = `<li>Could not load appointments.</li>`;
    return;
  }

  const rows = await res.json();
  renderAppointments(rows);
}

function renderAppointments(rows) {
  const list = document.getElementById("appointments-list");
  if (!rows || rows.length === 0) {
    list.innerHTML = `<li>No appointments yet.</li>`;
    return;
  }

  list.innerHTML = rows.map(r => `
    <li>
      <strong>${escapeHTML(r.patient_name || "Unknown")}</strong>
      <div>Date of Birth: ${escapeHTML(prettyDate(r.patient_dob))}</div>
      <div>Telephone: ${escapeHTML(r.patient_telephone || "-")}</div>
      <div>Address: ${escapeHTML(r.patient_address || "-")}</div>
      <div>Medical History: ${escapeHTML(r.patient_medical_history || "-")}</div>
      <div>Next of Kin: ${escapeHTML(r.patient_next_of_kin || "-")}</div>
      <div style="margin-top:6px; color:#5c6b7a; font-size:0.9rem;">Created: ${escapeHTML(prettyDateTime(r.created_at))}</div>
    </li>
  `).join("");
}

// Helpers
function escapeHTML(str) {
  return String(str || "")
    .replace(/&/g,"&amp;").replace(/</g,"&lt;").replace(/>/g,"&gt;");
}
function prettyDate(s) {
  if (!s) return "-";
  const d = new Date(s);
  return isNaN(d) ? s : d.toLocaleDateString("en-GB", { year:"numeric", month:"short", day:"2-digit" });
}
function prettyDateTime(s) {
  if (!s) return "-";
  const d = new Date(s);
  return isNaN(d) ? s : d.toLocaleString("en-GB", { year:"numeric", month:"short", day:"2-digit", hour:"2-digit", minute:"2-digit" });
}
```

## 🚀 Getting Started

### Prerequisites
- A Supabase project (**URL** + **publishable/anon key**).
- The table created and **RLS policies** configured.
- A browser or VS Code with **Live Server**.

### Setup
1. Clone or download this repository.
2. Update `static/app.js` with your Supabase `API_URL` and `API_KEY`.
3. Open `index.html` in your browser (or run with **Live Server**).
4. Use the form to create an appointment; check the list at the bottom.

**Insert (POST)**
```bash
curl -X POST \

  -H "Authorization: Bearer YOUR_PUBLISHABLE_KEY" \
  -H "Content-Type: application/json" \
  -d '{"patient_name":"Alex Smith","patient_dob":"2001-04-23","patient_telephone":"07123456789","patient_address":"123 High St, Shrewsbury SY1","patient_medical_history":"None","patient_next_of_kin":"Jamie Smith"}' \
```

**Select (GET)**
```bash
curl -H "apikey: YOUR_PUBLISHABLE_KEY" \
     -H "Authorization: Bearer YOUR_PUBLISHABLE_KEY" \
     "https://YOUR-PROJECT.supabase.co/rest/v1/tbl_customer_appointments?select=*&order=created_at.desc"
```

## 🔒 Security Notes

- The publishable (anon) key is acceptable for **public** REST calls **only** when **RLS policies** strictly govern access.
- For production, route writes through a backend using a **Service Role key** and stricter policies.

## 🛠️ Troubleshooting

- **Styles not loading?** Confirm the folder layout and exact link:
  ```html
  <link rel="stylesheet" href="static/style.css">
  ```
- **JS not loading?** Check DevTools → **Network** for `static/app.js` (status 200).
- **POST/GET failing?** Verify RLS policies and REST path casing (`tbl_customer_appointments`).
- **Dates**: `<input type="date">` sends `YYYY-MM-DD` and maps to Postgres `DATE`.
